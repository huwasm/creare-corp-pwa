Design Decisions

Q&A

Why a dedicated panel AND chat integration?

The dedicated panel is for the overview page — shows the full briefing when you click "View forecast" on a card. The chat version is for quick Q&A — same data, same logic, but inline with conversation. Both use the same 3-block computation.

Where do the scores come from?

Price Score: 60% ROC momentum + 40% ARIMA forecast, computed in JS from 30100_prices. Sentiment Score: decay-weighted FinBERT aggregation from 30200_news.finbert_* columns. Final Score: matrix lookup + flag modifier.

Why not just let the LLM decide?

The decision matrix is deterministic — same inputs always produce the same recommendation. The LLM only writes the narrative around an already-decided recommendation. This means the LLM can't hallucinate a "Buy" when the data says "Avoid".

What about GARCH?

GARCH estimates forward volatility. It doesn't affect the score — it sets the Flag (Stable/Watch/Alert). For hackathon, we can approximate with rolling std instead of full GARCH. The flag modifiers still work the same way.

Time windows?

ROC: 20-day price change. ARIMA: 30-day lookback, 30-day forecast. FinBERT: 30-day news window with exponential decay (today=1.0, 30d ago=0.22). GARCH: full history for fitting, 30-day forward variance forecast.

What does the user see?

Never sees scores, models, or technical terms. Just: a colored badge (Strong Buy / Buy / Hold / Avoid), a time suggestion ("Act within 24-48h"), and a 200-word briefing in plain English. The badges row is optional — for power users who want to see the breakdown.
