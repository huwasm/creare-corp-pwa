1
Layer 1 — Data Ingestion
✓ yfinance — free commodity price data (OHLCV), no API key
✓ NewsAPI — free tier 100 req/day, commodity news by keyword
✓ pandas — data wrangling, rolling calcs, time series
✓ Cache with st.cache_data or JSON file (avoid re-fetching in demo)
Fallback: If NewsAPI rate-limits, pre-cache 20 articles as JSON fixture. Demo still works.
2
Layer 2 — Signal Engine (Python)
✓ numpy + pandas — trend (linregress), volatility (rolling std), momentum
✓ Regime classifier: slope + volatility thresholds → bullish/bearish/sideways/volatile
✓ Spike detection: Z-score > 2σ on daily returns
✓ News: keyword scoring + Claude mini-call for sentiment batch
✓ Threat ranking: severity = sentiment × recency × category weight
✓ Forecast: simple linear extrapolation + range from volatility bands
Output: signals.json with price_features, news_features, threats[], forecast{}
3
Layer 3 — Claude Intelligence
✓ anthropic Python SDK — Claude Sonnet 4 (fast + cheap for hackathon)
✓ Structured output via tool_use — force JSON schema compliance
✓ Role-aware system prompt: Buyer framing vs Trader framing
✓ Output schema: {decision, entry, target, stop, confidence, why, risks{}, scenarios[], watchlist[]}
✓ Decision forced to exactly one of: BUY / SELL / WAIT
Key: Claude sees signals.json, NOT raw data. It reasons on pre-computed features.
4
Layer 4 — UI (Streamlit PWA)
✓ streamlit — fastest path to polished demo, zero frontend build
✓ plotly — interactive charts with forecast zone + event markers
✓ Custom CSS via st.markdown(unsafe_allow_html=True) for dark theme
✓ Mobile responsive via Streamlit columns + media queries in custom CSS
✓ Sections: Decision (hero) → Threats → Signals → News → Forecast
Deploy: streamlit run app.py — shareable URL via Streamlit Cloud (free)

Design Decisions
Q: Why structured signals before Claude?
CRITICAL — Without pre-computed features, judges see "prompt wrapper." The Signal Engine makes Claude a decision analyst, not a summarizer.
Q: Why BUY / SELL / WAIT instead of nuanced text?
CLARITY — Analysts need a clear signal. The big green/red/yellow box is the first thing they see. Details (entry, target, stop, why, risks) are below for context. One glance = one decision.
Q: Why a Threat Ranking?
GLOBAL AWARENESS — Raw news is noise. Ranked threats show what actually moves the market, sorted by severity. Analysts can scan top threats in 3 seconds instead of reading 8 articles.
Q: Why Forecast in the Signal Engine?
CREDIBILITY — A dashboard without "what's next" is just a report. The forecast (7D/30D target + range) gives the analyst a reference point. Even a simple linear extrapolation + volatility bands is more useful than nothing.
Q: Why Streamlit, not Next.js or React?
SPEED — 24h hackathon. Streamlit = 0 build step, built-in dark theme, native Plotly charts, instant deploy. You'd spend 8h on React boilerplate. Streamlit gets you to a polished demo in 4h of UI work.
Q: Mobile PWA with Streamlit?
WORKS — Streamlit is responsive out of the box. Use columns + custom CSS media queries for mobile. Tab-based nav on small screens. Not native-quality but very demo-able on a phone.
Q: Interactive news — how?
EXPANDERS — Streamlit st.expander() for each news item. Click to see impact analysis + signal direction. In this mockup it's onclick toggle. Same pattern.
Hackathon MVP scope: Signal Engine + Claude JSON output + BUY/SELL/WAIT decision card + Threat ranking are NON-NEGOTIABLE. Forecast and interactive news are high-value extras. Event markers on chart are nice-to-have.
Demo tip: Pre-cache one commodity (Copper) with real data. During demo, show the live flow once, then switch to cached data for reliability. Judges care about the system, not whether your API call works live.
